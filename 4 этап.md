## 1. Реализация лидербордов (рейтингов)

### Что нужно реализовать

1. **Типы рейтингов:**
    - по физическим очкам (движение, упражнения, скорость),
    - по промт-очкам (качество, структура, уникальность),
    - интегральный рейтинг (взвешенная сумма).
2. **Временные срезы:**
    - за тренировку / событие,
    - за день / неделю / месяц,
    - за сезон / учебный период.
3. **Фильтры и сегментация:**
    - глобальный рейтинг,
    - друзья,
    - группа / класс / школа,
    - регион / город.
4. **UX:**
    - позиция пользователя,    
    - подсветка роста / падения,
    - быстрый переход в профиль участника (если разрешено).
Обновление по завершению попытки + при открытии лидерборда.

## 2. Социальные функции (лайки, челленджи, публикации)

### Что нужно реализовать

1. **Социальная лента (минимальная):**
    - события друзей,
    - результаты челленджей,
    - достижения.
2. **Лайки и реакции:**
    - лайк результата тренировки,
    - лайк челленджа / достижения,
    - счётчик и список реакций (с ограничением видимости).
3. **Челленджи друзьям:**
    - вызов 1-на-1 или группе,
    - параметры: маршрут / время / очки,
    - дедлайн и статус (принят / завершён / проигран).
4. **Публикация результатов:**
    - публикация в ленту внутри приложения,
    - явное согласие пользователя (opt-in),
    - настройка приватности:
        - только я,
        - друзья,
        - группа/школа.

## 3. Магазин, экономика и NFT

### Что нужно реализовать

1. **Экономика:**
    - виртуальная валюта,
    - XP,
    - достижения / бейджи.
2. **Начисление:**
    - за прохождение маршрутов,
    - за место в рейтинге,
    - за челленджи,
    - за активность (daily/weekly streak).
3. **Магазин:**
    - косметические скины,
    - визуальные усиления (не влияющие на честность),
    - новые маршруты / темы.
4. **NFT-интеграция:**
    - кастодиальные NFT (без кошельков у детей)
    - SDK Thirdweb / Enjin
    - mint по событию (победа, достижение)
    - отображение инвентаря.
### Ограничения (важно)
- NFT **не должны давать геймплейное преимущество**.
- Все транзакции — через backend, не напрямую из клиента.

## 4. Дашборд тренера (React)

### Что нужно реализовать

1. **Онлайн-мониторинг:**
    - список активных событий,
    - статусы участников,
    - карта с GPS-треками.
2. **Протоколы и отчёты:**
    - тайминги входа в радиус,
    - QR-сканы,
    - выполненные упражнения,
    - промт-тексты и оценки.
3. **Управление контентом:**
    - шаблоны заданий,
    - rulesets промтов,
    - параметры маршрутов.
4. **Экспорт:**
    - XLS / CSV (аналитика),
    - PDF (официальные протоколы).
