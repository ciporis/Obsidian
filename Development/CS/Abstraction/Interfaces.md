Интерфейс - механизм, для определения правил поведения объектов, в них входят прототипы методов, свойств, индексаторов и событий. В отличие от классов в интерфейсах нельзя определять методы (с C# 6 можно, но это нарушает саму суть интерфейсов). 
В отличие от наследования классов базовых интерфейсов может быть любое количество

В теле интерфейса могут быть:
1. Метод
2. Свойство
3. Индексатор
4. Событие

Ни один из членов интерфейса не содержит оператора, определяющего действия
Intefrace - link type
Нельзя создать instance интерфейса
Очень удобно использовать объекты, наследуемые от интерфейса чтобы вызывать какие-либо методы, общие для классов, наследуемых от этого интерфейса

Если класс наследуется от нескольких интерфейсов, он обязан реализовать структуру каждого.
